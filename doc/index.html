Noze

Noze is a new programming language. Some interesting features are:

  • Safe: Noze functions can't access anything that wasn't passed in as a parameter. That means no global state, and no unsafe memory operations. They also can't perform I/O without permission being passed in by a parameter. This means that if you're tracking down a bug, you can exclude a function (and all code in it) based on just the arguments passed. Unsafe code (such as calling C libraries) is possible but can't be accidentally called from safe code.

    A strong notion of data; immutable, serializable types that are just "plain old data".

    The compiler allows data to be accessed in parallel (as well as some special types like fut), but not mutable types.

    This allows for a launch function that launches a new parallel task and is guaranteed to only access things that are thread-safe.

  • Templates; everything is a template.

    Templates are checked abstractly, meaning you get errors at the definition instead of at use.

    A spec allows a template to refer to functions that it doesn't have in scope but the caller does.

Types

Here is a complete list of every type (or kind of type). We'll go into more detail later.
  • record: Stores several named values.
  • union: Stores one of several other types. These are discriminated unions, meaning it knows which of the possibilities it has.
  • fun, fun-mut, fun-ref: Known as closures or lambdas in other languages.
  • nat, nat32, nat16, nat8, int, int32, int16, int8: Natural numbers and integers.
  • float and float32
  • bool, char, and void
  • ptr: C-style pointer for low level code
  • extern-ptr and fun-ptr for C compatibility.
