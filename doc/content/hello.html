Crow syntax is based on indented blocks. Here there is an indented block of code under import and one under main. The indentation is a mandatory part of the syntax. An indent may be one tab, two spaces, or four spaces; but every indent in the same file must use the same style.

Here's what each part does:
  • import imports the function print from the builtin module io.print. Other names like fut are implicitly imported due to being very common.
  • main defines a function. In crow, the name of a declaration always comes first. Then the return type fut exit-code comes. This instantiates the templated type fut. Then there is a parameter (named _) and its type arr str. Then there is a spec: summon. We won't get to specs until later. The spec allows us to call print which relies on it. (There are ways to do I/O without using summon.)
  • print: "Hello, world!" and 0 resolved are both function calls. (The functions are print and resolved.) Crow provides multiple syntaxes for calling functions; the semantics are the same. The final line is the function result. (0 is a nat and resolved wraps it in a fut.)
