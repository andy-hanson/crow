Variants

A variant is like a union, except its members can be declared anywhere.
So, unlike for a union, you don't know all the members of a variant, since other unrelated modules could add new members.

main void() info log "{15.verified-id is-adult}" info log "{60.verified-id is-adult}" info log "{no-id is-adult}" age variant # These could even go in a different module. verified-id variant-member(age, nat) no-id variant-member(age) is-adult bool(a age) match a as verified-id x x >= 18 as no-id false # Required for variants (since we don't know if other members might exist) else false

As with a union, there is a getter function for each member that returns an option.

main void() a v = 7 m if x ?= a m info log "{x}" () v variant m variant-member(v, nat)

You should generally prefer to use a union instead of a variant.
They were added to support letting users define custom exceptions.
