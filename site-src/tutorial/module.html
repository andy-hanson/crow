Modules

Each .crow file is a module. Modules can access each other only through imports.

An import like io.print references a builtin module. These come from the include directory that comes with Crow.

To import a module of your own, consider the relative path from the importing module to the imported module.
The relative path is specified by climbing up and then down. Start with a single ., then an additional . for each parent directory you want to climb. Then, write the name of each directory go down into, separated by ..

If you have these files:
  • root/a.crow
  • root/b.crow
  • root/x/c.crow
  • root/y/z/d.crow
If a wants to import:
  • b: Use .b
  • c: Use .x.c
  • d: Use .y.z.d
If d wants to import:
  • a: Use ...a
  • b: Use ...b
  • c: Use ...x.c

Circular imports between modules are not allowed, meaning if a imports b, b can't import a (and can't import any module that does).

Privacy

To make a definition private to its module, prepend a . to its name.

In the above example, other modules could import f but not g. Functions, specs, and types can all be made private.
